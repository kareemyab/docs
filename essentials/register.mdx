## What This Endpoint Does

<Steps>
  <Step title="Input Validation">
    Validates wallet address, content hash, and required parameters
  </Step>
  <Step title="IPFS Upload">
    Uploads file metadata to IPFS and generates Content Identifier (CID)
  </Step>
  <Step title="Identity Check">
    Automatically checks for recent identity verification and includes verified data if available
  </Step>
  <Step title="Blockchain Registration">
    Creates immutable proof record on Solana blockchain (signed automatically for managed wallets, or returns transaction for self wallets)
  </Step>
</Steps>

---

## Wallet Type Flows

<Card>
<Tabs>
  <Tab title='Managed ("managed")'>
    **For Trust Engine-created wallets only**
    
    Use `"walletType": "managed"` only for wallets created via [`/create-wallet`](/essentials/create-wallet)
    
    ✅ **Immediate completion** - Returns `200` with confirmed transaction  
    ✅ **No user action needed** - Trust Engine handles all signing  
    ✅ **Instant result** - Registration is complete when API call returns
  </Tab>
  
  <Tab title='Self with Action Link ("self")'>
    **For user's own wallets (Phantom, Solflare, etc.) - Recommended**
    
    Use `"walletType": "self"` + `"returnActionLink": true`
    
    ✅ **User-friendly** - Returns `202` with secure action link  
    ✅ **Browser compatible** - Works with all major wallet extensions  
    ✅ **Secure signing** - User signs via familiar wallet interface

    ### How It Works

    <Steps>
      <Step title="Set returnActionLink: true">
        API returns secure action link instead of raw transaction data
      </Step>
      <Step title="User visits link">
        User signs transaction using their browser wallet (Phantom, Solflare, etc.)
      </Step>
      <Step title="Auto-completion">
        Transaction is automatically submitted and registration completes
      </Step>
    </Steps>

    **Benefits**: Secure, user-friendly, browser-compatible, and links expire after 24 hours.
  </Tab>
  
  <Tab title='Self with Raw Transaction ("self")'>
    **For user's own wallets - Advanced integration**
    
    Use `"walletType": "self"` + `"returnActionLink": false`
    
    ⚙️ **Advanced** - Returns `202` with unsigned transaction data  
    ⚙️ **Manual handling** - You handle wallet connection and signing  
    ⚙️ **Submit required** - Use [`/transactions`](/essentials/submit-transaction) after signing
  </Tab>
</Tabs>
</Card>
