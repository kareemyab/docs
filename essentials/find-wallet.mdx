## What This Endpoint Does

<Steps>
    <Step title="Parameter Validation">
        Ensures that exactly one of `userID` or `walletAddress` is provided.
    </Step>
    <Step title="On-Chain Search">
        Queries the Solana blockchain to find the relationship Program Derived Address (PDA).
    </Step>
    <Step title="Fetches Relation">
        Retrieves and deserializes the on-chain data for the user-key relationship.
    </Step>
</Steps>

---

## Parameters

<ResponseField name="userID" type="string" optional>
  User ID to search for (provide either this OR walletAddress)
  
  **Constraints:** Cannot be used with walletAddress
  
  Example: `"user-12345"`
</ResponseField>

<ResponseField name="walletAddress" type="string" optional>
  Wallet address to search for (provide either this OR userID)
  
  **Constraints:** Must be a valid Solana public key, cannot be used with userID
  
  Example: `"9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"`
</ResponseField>

<Note>
You must provide exactly one of `userID` or `walletAddress` (not both).
</Note>

---



## Response Examples

### Success Response (200)

<ResponseField name="success" type="boolean">
  If it was successfull
  
  Example: `true`
</ResponseField>

<ResponseField name="message" type="string">
  Success confirmation message
  
  Example: `"Wallet relation found successfully using userID."`
</ResponseField>

<ResponseField name="details" type="object">
  Search and relationship details
  
  <Expandable title="Details Object">
    <ResponseField name="searchType" type="string">
      The type of search performed
      
      Example: `"userID"` or `"walletAddress"`
    </ResponseField>
    
    <ResponseField name="searchValue" type="string">
      The value that was searched for
      
      Example: `"user-12345"`
    </ResponseField>
    
    <ResponseField name="relation" type="object">
      The found relationship details
      
      <Expandable title="Relation Object">
        <ResponseField name="userId" type="string">
          The user ID in the relationship
          
          Example: `"user-12345"`
        </ResponseField>
        
        <ResponseField name="walletAddress" type="string">
          The wallet address in the relationship
          
          Example: `"9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"`
        </ResponseField>
        
        <ResponseField name="createdAt" type="string">
          ISO timestamp when the relationship was created
          
          Example: `"2024-01-15T10:30:45.123Z"`
        </ResponseField>
        
        <ResponseField name="relationPDA" type="string">
          Program Derived Address for this relationship
          
          Example: `"xyz789abc123def456ghi..."`
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>
