## Quick Start

### Installation

```bash
npm install trust-engine-sdk
```

### Complete Example

<CodeGroup>
```typescript SDK (Recommended)
import { getWalletBalance } from 'trust-engine-sdk';

async function checkBalance() {
  try {
    const result = await getWalletBalance({
      walletAddress: '9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM'
    });
    
    console.log('✅ Balance retrieved successfully');
    console.log(`Wallet: ${result.details.walletAddress}`);
    console.log(`Balance: ${result.details.balanceSOL} SOL`);
    console.log(`Lamports: ${result.details.balanceLamports.toLocaleString()}`);
    console.log(`Network: ${result.details.chain}`);
    
    return result;
  } catch (error) {
    console.error('❌ Failed to get balance:', error.message);
    throw error;
  }
}

// Usage
checkBalance()
  .then(result => console.log('Balance check completed:', result))
  .catch(error => console.error('Error:', error));
```

```bash cURL
curl -X POST http://localhost:3000/wallet-balance \
  -H "Content-Type: application/json" \
  -d '{
    "walletAddress": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"
  }'
```

```javascript Raw JavaScript
async function checkWalletBalance(walletAddress) {
  try {
    const response = await fetch('http://localhost:3000/wallet-balance', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ walletAddress })
    });
    
    const data = await response.json();
    
    if (response.ok) {
      console.log(`✅ Balance: ${data.details.balanceSOL} SOL`);
      console.log(`Lamports: ${data.details.balanceLamports.toLocaleString()}`);
      return data;
    } else {
      throw new Error(data.message);
    }
  } catch (error) {
    console.error('❌ Error checking balance:', error.message);
    throw error;
  }
}

// Usage
checkWalletBalance('9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM')
  .then(result => console.log('Wallet details:', result.details))
  .catch(error => console.error('Failed to get balance:', error));
```
</CodeGroup>

## Parameters

<ResponseField name="walletAddress" type="string" required>
  Valid Solana wallet address in Base58 format
  
  **Constraints:** Must be a valid Solana public key (32-44 characters)
  
  Example: `"9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"`
</ResponseField>

---

## Response Examples

### Success Response (200)

<ResponseField name="message" type="string">
  Success confirmation message
  
  Example: `"Wallet balance retrieved successfully."`
</ResponseField>

<ResponseField name="details" type="object">
  Wallet balance details
  
  <Expandable title="Details Object">
    <ResponseField name="walletAddress" type="string">
      The queried Solana wallet address
      
      Example: `"9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"`
    </ResponseField>
    
    <ResponseField name="balanceSOL" type="number">

      Balance in lamports (smallest unit, integer)
      
      Example: `1234567890`
    </ResponseField>
    
    <ResponseField name="chain" type="string">
      Network identifier showing which Solana cluster
      
      Example: `"solana-devnet"`
    </ResponseField>
  </Expandable>
</ResponseField>

## What This Endpoint Does

<Steps>
  <Step title="Address Validation">
    Validates the provided wallet address format using Solana's Base58 encoding
  </Step>
  <Step title="Balance Query">
    Queries the Solana blockchain via RPC for the current SOL balance
  </Step>
  <Step title="Unit Conversion">
    Converts and returns balance in both SOL and lamports for convenience
  </Step>
  <Step title="Network Info">
    Includes current network information for context
  </Step>
</Steps>

## Understanding Solana Balances

### SOL vs Lamports
<Info>
**1 SOL = 1,000,000,000 lamports**

- **SOL**: Human-readable format (e.g., 1.5 SOL)
- **Lamports**: Smallest unit, used for precise calculations
</Info>

### Balance Types
This endpoint returns **native SOL balance only**. For SPL token balances (like Reward Tokens), the system uses internal `checkUserTokenBalance` functions during operations like file registration.

### Network Information
The response includes network information in the `chain` field:

| Network | Chain Value | Description |
|---------|-------------|-------------|
| **Devnet** | `"solana-devnet"` | Development network for testing |
| **Testnet** | `"solana-testnet"` | Public testing network |
| **Mainnet** | `"solana-mainnet-beta"` | Production network |
| **Localnet** | `"solana-localnet"` | Local development cluster |
