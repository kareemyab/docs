<Info>
**Time to Complete**: ~5 minutes  
**What You'll Learn**: How to use Trust Engine CLI for wallet creation, file registration, and verification directly from your terminal
</Info>

<Steps>
<Step title="Install the Trust Engine CLI">
Install the CLI globally to use it from anywhere in your terminal
    
```bash
npm install -g @trust-engine/cli
```

**Verify installation:**
```bash
trust-engine-cli --version
```

**Get help and see all available commands:**
```bash
trust-engine-cli --help
```

**Expected output:**
```
Trust Engine CLI v1.0.0

Commands:
  create-wallet    Create a new wallet for a user
  find-wallet      Find a wallet by user ID or wallet address
  link-wallet      Link a wallet to a user ID
  register         Register a file with the Trust Engine
  search-file      Search for a file by its content hash
  submit-transaction Submit a signed transaction to the network

Options:
  --help     Show help
  --version  Show version number
  --json     Output results in JSON format
```

</Step>

<Step title="Create a Wallet">
Create a managed wallet that will own your registered files

<Warning>
**‚ö†Ô∏è ONE-TIME OPERATION**: This should only be performed **once per user**. The CLI will display the wallet address that you should save for future use.
</Warning>

```bash
# Note: userID must be unique across your entire application
trust-engine-cli create-wallet --userID "your-unique-user-id-123"
```

**With JSON output (for scripting):**
```bash

```

<Step title="Register a File">
Register any file from your computer on the blockchain

The CLI automatically handles file reading, hash calculation, and metadata extraction. Simply point to your file:

```bash
trust-engine-cli register \
  --file ./my-document.pdf \
  --contentTitle "My Important Document" \
  --walletAddress "YOUR_WALLET_ADDRESS_FROM_STEP_2" \
  --walletType "managed" \
  --metadata "Registered via CLI"
```

**Example with a real file:**
```bash
# Register a PDF document
trust-engine-cli register \
  --file ./contract.pdf \
  --contentTitle "Employment Contract" \
  --walletAddress "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM" \
  --walletType "managed" \
  --metadata "Legal document registration"
```

**Register with JSON output:**
```bash
trust-engine-cli register \
  --file ./my-document.pdf \
  --contentTitle "My Document" \
  --walletAddress "YOUR_WALLET_ADDRESS" \
  --walletType "managed" \
  --metadata "CLI registration" \
  --json
```

<Step title="Verify File Registration">
Search for your registered file to confirm it's on the blockchain

```bash

```

**Search with JSON output:**
```bash
trust-engine-cli search-file --file ./my-document.pdf --json
```

**Search by hash (if you have it):**
```bash
trust-engine-cli search-file --contentHash "a1b2c3d4e5f6789..."
```

## CLI Commands Reference

| Command | Purpose | Key Options |
|---------|---------|-------------|
| `create-wallet` | Create new managed wallet | `--userID` |
| `find-wallet` | Find wallet by user ID | `--userID`, `--walletAddress` |
| `link-wallet` | Link external wallet | `--userID`, `--walletAddress` |
| `register` | Register file on blockchain | `--file`, `--walletAddress`, `--contentTitle` |
| `search-file` | Search for registered files | `--file`, `--contentHash` |
| `submit-transaction` | Submit signed transaction | `--transaction`, `--walletAddress` |

## Global Options

| Option | Description |
|--------|-------------|
| `--json` | Output results in JSON format |
| `--help` | Show help for any command |
| `--version` | Show CLI version |
