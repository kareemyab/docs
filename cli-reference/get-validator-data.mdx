# get-validator-data

Get validator data and information for a specific wallet address.

## Usage

```bash
core-api-cli get-validator-data [options]
```

## Required Options

<ParamField path="--walletAddress, -w" type="string" required>
  Wallet address of the validator to query
</ParamField>

## Optional Options

<ParamField path="--json" type="boolean">
  Output the result as JSON instead of formatted text
</ParamField>

## Examples

### Get Validator Data

```bash
core-api-cli get-validator-data \
  --walletAddress "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"
```

### JSON Output for Automation

```bash
core-api-cli get-validator-data \
  --walletAddress "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM" \
  --json
```

## Error Handling

Common error scenarios:

- **Invalid wallet address**: Ensure the wallet address is a valid Solana address
- **Validator not found**: The wallet address may not be associated with a validator
- **Network issues**: Check your internet connection and blockchain RPC availability
- **Inactive validator**: The validator may be delinquent or not currently validating

## Use Cases

### Monitor Validator Performance

```bash
#!/bin/bash
validator_address="9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"

result=$(core-api-cli get-validator-data \
  --walletAddress "$validator_address" \
  --json)

status=$(echo "$result" | jq -r '.details.status')
commission=$(echo "$result" | jq -r '.details.commission')

echo "Validator Status: $status"
echo "Commission Rate: $commission"

if [ "$status" = "Active" ]; then
  echo "✅ Validator is active and validating"
else
  echo "❌ Validator is not active"
fi
```

### Validator Comparison

```bash
#!/bin/bash
validators=("validator1..." "validator2..." "validator3...")

echo "Validator Performance Comparison:"
echo "=================================="

for validator in "${validators[@]}"; do
  echo "Checking validator: $validator"
  
  result=$(core-api-cli get-validator-data \
    --walletAddress "$validator" \
    --json 2>/dev/null)
  
  if [ $? -eq 0 ]; then
    status=$(echo "$result" | jq -r '.details.status')
    commission=$(echo "$result" | jq -r '.details.commission')
    stake=$(echo "$result" | jq -r '.details.totalStake')
    
    echo "  Status: $status"
    echo "  Commission: $commission"
    echo "  Total Stake: $stake"
  else
    echo "  ❌ Failed to retrieve data"
  fi
  echo
done
```

### Stake Pool Analysis

```bash
#!/bin/bash
stake_pool_validators_file="validators.txt"

if [ ! -f "$stake_pool_validators_file" ]; then
  echo "Create a file named 'validators.txt' with one validator address per line"
  exit 1
fi

total_stake=0
active_validators=0

while IFS= read -r validator; do
  result=$(core-api-cli get-validator-data \
    --walletAddress "$validator" \
    --json 2>/dev/null)
  
  if [ $? -eq 0 ]; then
    status=$(echo "$result" | jq -r '.details.status')
    
    if [ "$status" = "Active" ]; then
      ((active_validators++))
      stake=$(echo "$result" | jq -r '.details.totalStake' | sed 's/ SOL//')
      total_stake=$(echo "$total_stake + $stake" | bc)
    fi
  fi
done < "$stake_pool_validators_file"

echo "Stake Pool Summary:"
echo "Active Validators: $active_validators"
echo "Total Stake: $total_stake SOL"
```

## Notes

- Validator data is retrieved from the Solana blockchain
- Information includes both current status and historical performance metrics
- Commission rates and stake amounts may change over time
- Vote account information shows the validator's participation in consensus
- Epoch credits indicate the validator's performance in recent epochs
- This data is useful for stake pool operators and delegators making informed decisions
