## Step 1: Create a Wallet

First, create a Solana wallet that will be linked to your user ID. This wallet will be used for all your provenance registrations.

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/create-wallet \
  -H "Content-Type: application/json" \
  -d '{
    "userID": "quickstart_user_001"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/create-wallet', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ 
    userID: 'quickstart_user_001' 
  })
});

const walletData = await response.json();
console.log('Wallet created:', walletData.details.walletAddress);
```
</CodeGroup>

### Expected Response

```json

  "message": "Success! Your Crossmint wallet has been created and linked to your user ID.",
  "details": {

  }
```

<Tip>
**Save your wallet address!** You'll need it for the next steps. The wallet is automatically funded with tokens needed for registration fees.
</Tip>

## Step 2: Register Your First File

Now register a digital media file to establish its provenance. The file content is never exposed - only a cryptographic hash is used.

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/register \
  -F "file=@/path/to/your/document.pdf" \
  -F "walletAddress=9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM" \
  -F "contentTitle=My First Provenance Record" \
  -F "contentDescription=Testing Trust Engine's provenance verification" \
  -F "walletType=crossmint" \
  -F 'metadata={"author":"Quickstart User","category":"test"}'
```

```javascript JavaScript
const formData = new FormData();
formData.append('file', fileInput.files[0]); // Your file input
formData.append('walletAddress', '9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM');
formData.append('contentTitle', 'My First Provenance Record');
formData.append('contentDescription', 'Testing Trust Engine\'s provenance verification');
formData.append('walletType', 'crossmint');
formData.append('metadata', JSON.stringify({
  author: 'Quickstart User',
  category: 'test'
}));

const response = await fetch('http://localhost:3000/register', {
  method: 'POST',
  body: formData
});

const registrationData = await response.text();
console.log('Registration result:', registrationData);
```
</CodeGroup>

## Step 3: Verify File Provenance

Search for your registered file to confirm its provenance record exists and check verification status.

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/search \
  -F "file=@/path/to/your/document.pdf"
```

```javascript JavaScript
const searchForm = new FormData();
searchForm.append('file', fileInput.files[0]); // Same file as before

const response = await fetch('http://localhost:3000/search', {
  method: 'POST',
  body: searchForm
});

const searchResults = await response.json();
console.log('Provenance records found:', searchResults.details.length);
```
</CodeGroup>

## Step 4: Check Your Wallet Balance

Monitor your wallet's token balance to ensure you have sufficient funds for future registrations.

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/wallet-balance \
  -H "Content-Type: application/json" \
  -d '{
    "walletAddress": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/wallet-balance', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    walletAddress: '9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM'
  })
});

const balanceData = await response.json();
console.log(`Balance: ${balanceData.details.balanceSOL} SOL`);
```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Server Connection Issues">
    **Problem**: `Connection refused` or timeout errors
    
    **Solutions**:
    - Ensure the Trust Engine server is running on port 3000
    - Check your network connectivity
    - Verify the server URL is correct for your environment
  </Accordion>

  <Accordion title="File Upload Failures">
    **Problem**: File type not supported or size limit exceeded
    
    **Solutions**:
    - Use supported file types (PDF, images, documents, etc.)
    - Ensure files are under 50MB
    - Check that files aren't corrupted or password-protected
  </Accordion>

  <Accordion title="Wallet Issues">
    **Problem**: Insufficient balance or wallet errors
    
    **Solutions**:
    - Check wallet balance using the `/wallet-balance` endpoint
    - Ensure your user ID is correctly linked to the wallet
    - Contact support if wallet creation fails
  </Accordion>
</AccordionGroup>
