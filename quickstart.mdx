## Step 1: Create Your Wallet
<Warning>
**‚ö†Ô∏è ONE-TIME OPERATION**: This step should only be performed **once per user**. The generated wallet will be used for **all your future file registrations**. If you ever forget your wallet address, you can always use your user ID to remember it using our API!
</Warning>

Create a managed wallet that will own your registered files:

<CodeGroup>
```bash cURL
curl -X POST https://core-api-server.onrender.com/create-wallet \
  -H "Content-Type: application/json" \
  -d '{"userID": "my-unique-id-123"}'
```

```javascript JavaScript (SDK)
const wallet = await sdk.createWallet({
  userID: 'my-unique-id-123'
});

console.log('Wallet created:', wallet.details.walletAddress);
const walletAddress = wallet.details.walletAddress;
```

```bash CLI
trust-engine-cli create-wallet \
  --userID "my-unique-id-123"
```
</CodeGroup>

**Expected Response:**
```json
{
  "message": "Success! Your managed wallet has been created and linked to your user ID.",
  "details": {
    "walletAddress": "9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM",
    "chain": "solana",
    "transactionSignature": "5J7X...",
    "explorerUrl": "https://explorer.solana.com/tx/5J7X...?cluster=devnet",
    "emailLinkedToWallet": "hadi@trustengine.org"
  }
}
```

**üíæ SAVE THIS INFORMATION**: Copy the `walletAddress` from the response and save it in a text file or notepad. You'll need to paste this exact address into Step 4 to register your file!

<Tip>
**Recommended**: Create a file called `trust-engine-info.txt` and save:
```
Wallet Address: 9WzDXwBbmkg8ZTbNMqUxvQRAyrZzDsGYdLVL9zYtAWWM
User ID: my-unique-id-123
```
</Tip>

## Step 2: Prepare Your File

For this example, let's register a simple text file. Create the file first:

<CodeGroup>
```bash cURL
# Create example file
echo "Hello, Trust Engine!" > example.txt

# Calculate SHA256 hash
shasum -a 256 example.txt
# Output: a7b2c3d4e5f6... example.txt

# Get file size
wc -c < example.txt
# Output: 21

# üíæ SAVE THE HASH: Copy the hash value for Step 4!
# Add this to your trust-engine-info.txt:
# File Hash: a7b2c3d4e5f6...
# File Size: 21
```

```javascript JavaScript (SDK)
import { CoreApiSDK } from '@trust-engine/sdk';

const sdk = new CoreApiSDK();

// Create example file (in browser, this would be from file input)
const fileContent = "Hello, Trust Engine!";
const file = new File([fileContent], "example.txt", { type: "text/plain" });

console.log('File ready for registration');
```

```bash CLI
# Create example file
echo "Hello, Trust Engine!" > example.txt

# The CLI will automatically handle hash calculation
ls -la example.txt
```
</CodeGroup>

<Info>
**üíæ IMPORTANT**: If using cURL method, copy the hash value (the long string before `example.txt`) to your `trust-engine-info.txt` file. You'll need this exact hash for Step 4. For SDK/CLI users, this is handled automatically.
</Info>

## Step 3: Register Your File

Now we'll combine everything from the previous steps to register your file on the blockchain. You'll need:
- **Wallet Address** from Step 2
- **File Hash** from Step 3  
- **File Size** from Step 3

Replace the placeholder values below with your actual data:

<CodeGroup>
```bash cURL
curl -X POST https://core-api-server.onrender.com/register \
  -H "Content-Type: application/json" \
  -d '{
    "contentTitle": "My First Registration",
    "walletAddress": "YOUR_WALLET_ADDRESS_FROM_STEP_2",
    "walletType": "managed",
    "contentHash": "YOUR_FILE_HASH_FROM_STEP_3",
    "fileMetadata": {
      "fileName": "example.txt",
      "fileSize": YOUR_FILE_SIZE_FROM_STEP_3,
      "mimeType": "text/plain"
    },
    "metadata": "Quickstart tutorial example"
  }'
```

```javascript JavaScript (SDK)
const registration = await sdk.registerFile({
  file: file, // The File object from Step 3
  contentTitle: 'My First Registration',
  walletAddress: walletAddress, // From Step 2
  walletType: 'managed',
  metadata: 'Quickstart tutorial example'
});

console.log('File registered!', registration);
```

```bash CLI
# Register the file (replace with your wallet address from Step 2)
trust-engine-cli register \
  --file example.txt \
  --contentTitle "My First Registration" \
  --walletAddress "YOUR_WALLET_ADDRESS_FROM_STEP_2" \
  --walletType "managed" \
  --metadata "Quickstart tutorial example"
```
</CodeGroup>

<Info>
**Note**: We're using managed wallets which handle transaction signing automatically. For self wallets, such as in Metamask, Phantom, etc., additional signing steps are required, and is not available in the CLI.
</Info>

**Expected Response (Managed):**
```json
{
  "message": "‚úÖ Accepted! Your file has been submitted and is being processed by the managed wallet.",
  "details": {
    "status": "processing",
    "managedTransactionId": "abc123...",
    "contentRegistrationPDA": "xyz789...",
    "contentHash": "a7b2c3d4...",
    "ipfsCid": "QmExample..."
  }
}
```

## Step 4: Verify Your Registration

Search for your registered file to confirm it's in the system. Use the same **file hash** and **wallet address** from previous steps:

<CodeGroup>
```bash cURL
curl -X POST https://core-api-server.onrender.com/search \
  -H "Content-Type: application/json" \
  -d '{
    "contentHash": "YOUR_FILE_HASH_FROM_STEP_3", 
    "walletAddress": "YOUR_WALLET_ADDRESS_FROM_STEP_2"
  }'
```

```javascript JavaScript (SDK)
const results = await sdk.searchFile(file); // Pass the same file from Step 3

console.log('Search results:', results);
```

```bash CLI
trust-engine-cli search-file \
  --file example.txt
```
</CodeGroup>

**Expected Response:**

## Step 5: Add TRUST Stamp Widget (Optional)

Now that you have registered content, you can add the TRUST Stamp widget to automatically display verification badges on your website:

<CodeGroup>
```html Basic Implementation
<script
  src="https://cdn.jsdelivr.net/npm/@trust-engine/widget@latest/dist/trust-engine-widget.js"
  data-api-base-url="https://core-api-server.onrender.com"
  data-enabled-elements='["img","video","audio"]'
  data-badge-position="bottom-right"
  data-theme="light"
  data-auto-scan="true"
  data-observe-dynamic-content="true"
  data-show-tooltip="true"
  data-debug="false"
></script>
```

```jsx React / Next.js
import Script from 'next/script';

export default function Layout({ children }) {
  return (
    <html>
      <head>
        <Script
          src="https://cdn.jsdelivr.net/npm/@trust-engine/widget@latest/dist/trust-engine-widget.js"
          data-api-base-url="https://core-api-server.onrender.com"
          data-enabled-elements='["img","video","audio"]'
          data-badge-position="bottom-right"
          data-theme="light"
          data-auto-scan="true"
          data-observe-dynamic-content="true"
          data-show-tooltip="true"
          data-debug="false"
          strategy="afterInteractive"
        />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

```vue Vue.js
<template>
  <div id="app">
    <!-- Your app content -->
  </div>
</template>

<script>
export default {
  mounted() {
    const script = document.createElement('script');
    script.src = "https://cdn.jsdelivr.net/npm/@trust-engine/widget@latest/dist/trust-engine-widget.js";
    script.setAttribute('data-api-base-url', 'https://core-api-server.onrender.com');
    script.setAttribute('data-enabled-elements', '["img","video","audio"]');
    script.setAttribute('data-badge-position', 'bottom-right');
    script.setAttribute('data-theme', 'light');
    script.setAttribute('data-auto-scan', 'true');
    script.setAttribute('data-observe-dynamic-content', 'true');
    script.setAttribute('data-show-tooltip', 'true');
    script.setAttribute('data-debug', 'false');
    document.head.appendChild(script);
  }
}
</script>
```
</CodeGroup>

The widget will automatically scan your website for media files and display verification badges on any registered content. When visitors click these badges, they'll see the complete provenance and registration details from your Trust Engine registration.

<Info>
**Complete Integration**: For more configuration options and detailed documentation, visit the [TRUST Stamp Widget](/trust-stamp) page.
</Info>
