## Step 1: Create a Wallet

First, create a Solana wallet that will be linked to your user ID. This wallet will be used for all your provenance registrations.

<CodeGroup>
```typescript SDK (Recommended)
import { createWallet } from 'trust-engine-sdk';

async function setupWallet() {
  try {
    const result = await createWallet({ 
      userID: 'quickstart_user_001' 
    });
    
    console.log('‚úÖ Wallet created successfully!');
    console.log('Wallet Address:', result.details.walletAddress);
    console.log('Transaction:', result.details.transactionSignature);
    
    // Save wallet address for next steps
    const walletAddress = result.details.walletAddress;
    return walletAddress;
  } catch (error) {
    console.error('‚ùå Wallet creation failed:', error.message);
    throw error;
  }
}

// Usage
const walletAddress = await setupWallet();
```

```bash cURL
curl -X POST https://core-api-server.onrender.com/create-wallet \
  -H "Content-Type: application/json" \
  -d '{
    "userID": "quickstart_user_001"
  }'
```

```javascript Raw JavaScript
const response = await fetch('https://core-api-server.onrender.com/create-wallet', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ 
    userID: 'quickstart_user_001' 
  })
});

const walletData = await response.json();
console.log('Wallet created:', walletData.details.walletAddress);
```
</CodeGroup>

## Step 2: Register Your First File

Now register a digital media file to establish its provenance. The file content is never exposed - only a cryptographic hash is used.

<Info>
**Two Registration Paths**: Trust Engine supports both **Crossmint wallets** (fully automated) and **standard wallets** (requires manual transaction signing). Choose the approach that matches your wallet type.
</Info>

<Tabs>
  <Tab title="Crossmint Wallet (Automated)">
    **Recommended for beginners** - The server handles transaction signing automatically.

    <CodeGroup>
    ```typescript SDK (Recommended)
    import { registerFile } from 'trust-engine-sdk';

    async function registerWithCrossmint(walletAddress: string, file: File) {
      try {
        const result = await registerFile({
          file: file, // File object from input or drag-drop
          contentTitle: 'My First Provenance Record',
          contentDescription: 'Testing Trust Engine\'s provenance verification',
          walletAddress: walletAddress,
          walletType: 'crossmint', // Automatic transaction handling
          metadata: {
            author: 'Quickstart User',
            category: 'test'
          }
        });
        
        console.log('‚úÖ File registered successfully!');
        console.log('Transaction ID:', result.details?.transactionSignature);
        console.log('IPFS CID:', result.details?.ipfsCid);
        console.log('Status:', result.message);
        
        return result;
      } catch (error) {
        console.error('‚ùå Registration failed:', error.message);
        throw error;
      }
    }

    // Usage (assuming you have a file from an input)
    ```

  <Tab title="Standard Wallet (Manual Signing)">
    **For advanced users** - You handle transaction signing with your own wallet (Phantom, Solflare, etc.).

    <CodeGroup>
    ```typescript SDK (Recommended)
    import { registerFile, submitTransaction } from 'trust-engine-sdk';

    async function registerWithStandardWallet(walletAddress: string, file: File) {
      try {
        // Step 1: Get prepared transaction from register endpoint
        const prepResult = await registerFile({
          file: file,
          contentTitle: 'My First Provenance Record',
          contentDescription: 'Testing Trust Engine\'s provenance verification',
          walletAddress: walletAddress,
          walletType: 'standard', // Returns unsigned transaction
          metadata: {
            author: 'Quickstart User',
            category: 'test'
          }
        });
        
        console.log('üìã Transaction prepared, ready for signing');
        console.log('IPFS CID:', prepResult.details?.ipfsCid);
        
        // Step 2: Sign transaction with your wallet
        // This would typically use a wallet adapter like @solana/wallet-adapter
        const base64Transaction = prepResult.details?.transaction; // From register response
        
        // Note: In a real app, you'd sign with wallet.signTransaction()
        // const signedTx = await wallet.signTransaction(transaction);
        // const base64Transaction = Buffer.from(signedTx.serialize()).toString('base64');
        
        // Step 3: Submit the signed transaction
        const submitResult = await submitTransaction({
          base64Transaction: base64Transaction
        });
        
        console.log('‚úÖ Transaction submitted and confirmed!');
        console.log('Transaction Signature:', submitResult.details.transactionSignature);
        console.log('Explorer URL:', submitResult.details.explorerUrl);
        
        return submitResult;
      } catch (error) {
        console.error('‚ùå Registration failed:', error.message);
        throw error;
      }
    }
    ```

    ```bash cURL (Two-Step Process)
    # Step 1: Get prepared transaction
    curl -X POST https://core-api-server.onrender.com/register \
      -F "file=@/path/to/your/document.pdf" \

      -F "contentDescription=Testing Trust Engine's provenance verification" \
      -F "walletType=standard" \
      -F 'metadata={"author":"Quickstart User","category":"test"}'

    # Step 2: Sign transaction with your wallet software, then submit
    curl -X POST https://core-api-server.onrender.com/submit-transaction \
      -H "Content-Type: application/json" \
      -d '{
        "transaction": "SIGNED_BASE64_TRANSACTION_FROM_STEP1"
      }'
    ```

    ```javascript Raw JavaScript
    async function registerWithStandardWallet() {
      // Step 1: Get prepared transaction
      const formData = new FormData();
      formData.append('file', fileInput.files[0]);
      formData.append('walletAddress', 'YOUR_WALLET_ADDRESS');
      formData.append('contentTitle', 'My First Provenance Record');
      formData.append('contentDescription', 'Testing Trust Engine\'s provenance verification');
      formData.append('walletType', 'standard');
      formData.append('metadata', JSON.stringify({
        author: 'Quickstart User',
        category: 'test'
      }));

      const prepResponse = await fetch('https://core-api-server.onrender.com/register', {
        method: 'POST',
        body: formData
      });
      
      const prepData = await prepResponse.text();
      console.log('Transaction prepared:', prepData);
      
      // Step 2: Extract base64 transaction and sign with your wallet
      // (In real implementation, parse the response to get the transaction)
      const base64Transaction = 'SIGNED_TRANSACTION_HERE'; // After wallet signing
      
      // Step 3: Submit signed transaction
      const submitResponse = await fetch('https://core-api-server.onrender.com/submit-transaction', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ transaction: base64Transaction })
      });
      
      const submitData = await submitResponse.json();
      console.log('Transaction confirmed:', submitData);
    }
    ```

## Step 3: Verify File Provenance

Search for your registered file to confirm its provenance record exists and check verification status.

<CodeGroup>
```typescript SDK (Recommended)
import { searchFile } from 'trust-engine-sdk';

async function verifyFileProvenance(file: File) {
  try {
    const result = await searchFile(file);
    
    if (result.details && result.details.length > 0) {
      console.log(`‚úÖ Found ${result.details.length} provenance record(s)!`);
      
      result.details.forEach((record, index) => {
        console.log(`\nüìÑ Record ${index + 1}:`);
        console.log(`   Creator: ${record.onChainRegistration.creator}`);
        console.log(`   IPFS CID: ${record.onChainRegistration.ipfsCid}`);
        console.log(`   Verification Status: ${record.onChainRegistration.finalized ? 'Finalized' : 'Pending'}`);
        console.log(`   Consensus: ${record.onChainRegistration.consensusPercentage}%`);
        console.log(`   User ID: ${record.userID || 'Not linked'}`);
      });
    } else {
      console.log('‚ÑπÔ∏è No provenance records found for this file');
    }
    
    return result;
  } catch (error) {
    if (error.message.includes('does not exist')) {
      console.log('‚ÑπÔ∏è File has not been registered yet');
    } else {
      console.error('‚ùå Search failed:', error.message);
    }
    throw error;
  }
}

// Usage
// await verifyFileProvenance(file);
```

### Understanding Verification Status

<Tabs>
  <Tab title="‚úÖ Verified">
    **Status**: `finalized: true` with ‚â•67% consensus
    
    Your file has been successfully verified by the network. The provenance is confirmed and can be trusted.
  </Tab>
  
  <Tab title="‚è≥ Pending">
    **Status**: `finalized: false`
    
    Verification is still in progress. Independent verifiers are reviewing your submission.
  </Tab>
  
  <Tab title="‚ùå Rejected">
    **Status**: `finalized: true` with &lt;67% consensus
    
    The network couldn't reach consensus on your file's authenticity. This is rare but can happen with corrupted or suspicious files.
  </Tab>
</Tabs>

## Step 4: Check Your Wallet Balance

Monitor your wallet's token balance to ensure you have sufficient funds for future registrations.

<CodeGroup>
```typescript SDK (Recommended)
import { getWalletBalance } from 'trust-engine-sdk';

async function checkWalletBalance(walletAddress: string) {
  try {
    const result = await getWalletBalance({ walletAddress });
    
    console.log('üí∞ Wallet Balance Information:');
    console.log(`   Address: ${result.details.walletAddress}`);
    console.log(`   Balance: ${result.details.balanceSOL} SOL`);
    console.log(`   Lamports: ${result.details.balanceLamports.toLocaleString()}`);
    console.log(`   Network: ${result.details.chain}`);
    
    // Check if balance is sufficient for registrations
    if (result.details.balanceSOL > 0.01) {
      console.log('‚úÖ Sufficient balance for file registrations');
    } else {
      console.log('‚ö†Ô∏è  Low balance - consider adding more SOL for registrations');
    }
    
    return result;
  } catch (error) {
    console.error('‚ùå Balance check failed:', error.message);
    throw error;
  }
}
```
